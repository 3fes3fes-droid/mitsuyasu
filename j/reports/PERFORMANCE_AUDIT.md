# 動作・性能監査

## 結論

旧サイトは壊れてはいませんでしたが、各話ページで全275話の詳細データを最初から読み込み、検索・選択のたびに一覧HTMLを全再生成していました。重く感じる原因として妥当です。

修正版では以下へ変更しました。

- 詳細データを54チャンクへ分割
- 初期表示では一覧用カタログと選択中のチャンクだけを読む
- 渋谷事変など長い区分も12話単位へ分割
- 検索時は一覧を作り直さず、既存行の表示・非表示だけを切り替える
- 選択時は一覧全体を再生成せず、選択行と詳細欄だけ更新する
- ページ別の軽量カタログを用意
- JavaScriptの読み込み順を `catalog → core → page` に統一

## 検査上の限界

この環境のChromiumはローカルファイルとlocalhostへの自動遷移が管理ポリシーで遮断されたため、今回の全件巡回はJSDOMで実施しています。JavaScript、DOM更新、動的script読込、検索、ID切替は検査済みです。実機Chromeでの最終体感確認は、解凍後の `index.html` または `python tools/serve.py` で行えます。
