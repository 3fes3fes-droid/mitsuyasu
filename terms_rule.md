# terms.json 追加ルール

## フォーマット
```json
{
  "category": "カテゴリ名",
  "term": "用語名",
  "desc": "説明文",
  "full": "英語正式名（英略語のときだけ。日本語用語はこの行ごと不要）",
  "related": ["関連用語1", "関連用語2"]
}
```

## カテゴリ（この22種類から選ぶ・新しく作らない）
アルゴリズムとプログラミング／コンピュータ構成要素／サービス管理／システム構成要素／システム監査／システム開発／ソフトウェア／データベース／ネットワーク／ビジネスシステム／プロジェクト管理／マーケティング／企業活動／基礎理論／情報セキュリティ／新技術・イノベーション／最新技術・その他／法務／経営戦略／設計手法／財務会計／開発手法

## fullのルール
- 英略語（RPA・AESなど）のときだけ書く
- 日本語用語（テレワーク・暗黙知など）は `"full"` の行ごと不要

## relatedのルール
- terms.json に実際に存在する用語名だけ書く
- なければ空配列 `[]` か省略

## 依頼のしかた
このメモと一緒に「〇〇という単語を追加して」と伝えるだけ。
Claudeはスニペット（上記フォーマットのJSON）だけ返す。
